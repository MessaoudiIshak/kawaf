## 1. CREATE BLOG POST (POST)

Endpoint: POST /api/blog
Auth Required: YES (USER, STAFF, ADMIN)

Request Headers:
{
  "Content-Type": "application/json",
  "Authorization": "Bearer {{TOKEN}}"
}

Request Body:
{
  "title": "Welcome to Kawaf Cat Café",
  "content": "We're excited to announce the opening of Kawaf, the premier cat café in the city. Here, you can enjoy delicious food and drinks while spending time with our adorable feline friends. Each cat available for adoption has its own unique personality and story.",
  "author": "Admin User",
  "photoUrl": "https://images.example.com/kawaf-cafe.jpg",
  "isPublished": true
}

Response (201 Created):
{
  "id": 1,
  "title": "Welcome to Kawaf Cat Café",
  "content": "We're excited to announce the opening of Kawaf...",
  "author": "Admin User",
  "photoUrl": "https://images.example.com/kawaf-cafe.jpg",
  "isPublished": true,
  "createdAt": "2026-02-25T10:30:00.000Z",
  "updatedAt": "2026-02-25T10:30:00.000Z"
}

## 9. ERROR SCENARIOS

### Missing Required Fields (POST)
Request Body:
{
  "title": "Missing Content Field",
  "author": "Admin"
  // Missing: content
}

Response (400 Bad Request):
{
  "error": "Missing required fields: title, content, and author"
}

---

### Invalid Blog Post ID
Endpoint: GET /api/blog/invalid-id

Response (400 Bad Request):
{
  "error": "Invalid blog post id"
}

---

### Unauthorized Access (No Token)
Endpoint: POST /api/blog
Without Authorization header

Response (401 Unauthorized):
{
  "error": "Unauthorized"
}

---

### Access Denied - Public User Cannot Edit
Request:
{
  "Authorization": "Bearer {{PUBLIC_TOKEN}}"
}

Note: Public users without USER/STAFF/ADMIN role cannot create, update, or delete

Response (401 Unauthorized):
{
  "error": "Unauthorized"
}

## 10. COMPLETE TEST WORKFLOW

Follow this sequence to test complete functionality:

1. Login as ADMIN
   POST /api/user/login
   Body: { "email": "admin@kawaf.com", "password": "password" }
   → Save token from response

2. Create Published Blog Post
   POST /api/blog
   Body: { "title": "...", "content": "...", "author": "...", "isPublished": true }
   → Save post ID (1)

3. Create Draft Blog Post
   POST /api/blog
   Body: { "title": "...", "content": "...", "author": "...", "isPublished": false }
   → Save post ID (2)

4. Get All Posts (as Admin)
   GET /api/blog
   → Should see 2 posts (published + draft)

5. Get All Posts (as Public, no token)
   GET /api/blog
   → Should see 1 post (only published)

6. Get Single Post
   GET /api/blog/1
   → Should return post details

7. Update Post
   PUT /api/blog/1
   Body: { "content": "Updated..." }
   → Should return updated post

8. Publish Draft
   PUT /api/blog/2
   Body: { "isPublished": true }
   → Should now be visible to public

9. Verify Public Visibility
   GET /api/blog (no token)
   → Should see both posts now

10. Delete Post
    DELETE /api/blog/2
    → Should return success message

11. Verify Deletion
    GET /api/blog/2
    → Should return 404 error

## 11. POSTMAN COLLECTION IMPORT

To import these tests into Postman:

1. Create a new Collection: "Kawaf Blog API Tests"

2. Set Collection Variables:
   - BASE_URL: http://localhost:3000
   - TOKEN: (will be set after login)

3. Create folder "Blog"

4. Add Requests:
   - Create Blog Post
   - Get All Posts
   - Get Single Post
   - Update Blog Post
   - Delete Blog Post
   - Publish Post
   - Unpublish Post

5. For each request, use:
   - {{BASE_URL}}/api/blog for collection interactions
   - {{TOKEN}} in Authorization header (Bearer)

6. Create Tests tab for automation:
   if (pm.response.code === 200 || pm.response.code === 201) {
     pm.test("Status code is 200/201", function() {
       pm.expect(pm.response.code).to.be.oneOf([200, 201]);
     });
   }

## 13. NOTES

- All timestamps are in UTC (ISO 8601 format)
- Content field supports plain text and markdown
- photoUrl is optional and can be null
- isPublished defaults to false for new posts
- Drafts are only visible to ADMIN and STAFF users
- Deleting a post permanently removes it from database
- Update operations support partial field updates
- Author field is required and cannot be null after creation
